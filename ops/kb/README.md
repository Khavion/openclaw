# ops/kb — Support knowledge base

One markdown file per topic, named `<topic-slug>.md`. The support-drafter
skill searches here before drafting any reply, and files new topics here
after a reply ships (G1-approved).

Format per file: a one-line summary, the canonical answer in plain customer
language, then internal notes (what not to promise, related tasks).

Empty at bootstrap: articles accumulate from real tickets.
